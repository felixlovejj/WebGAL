# 《代码中的中国：1024航迹云》开发部署教程

## 一、项目结构速览

```
packages/webgal/public/game/
├── background/      ← 背景图（jpg/png）
├── figure/          ← 角色立绘 & 小头像（png，透明背景）
├── bgm/             ← 背景音乐（mp3/ogg）
├── scene/           ← 剧本文件（.txt）★你改得最多的
├── vocal/           ← 语音（可选）
├── config.txt       ← 游戏标题等配置
└── template/        ← UI模板
```

一切修改都在 `packages/webgal/public/game/` 下进行，不需要动引擎代码。

---

## 二、本地预览（开发模式）

```bash
# 在项目根目录
yarn dev
```

打开 `http://localhost:3000/`，修改剧本或素材后刷新浏览器即可看到效果（Vite HMR，无需重启）。

---

## 三、构建上线版本

每次修改完要部署前，在**项目根目录**执行：

```bash
yarn webgal:build
```

这会在 `packages/webgal/dist/` 生成最终静态文件。里面包含了全部 JS/CSS/游戏素材。

---

## 四、部署到 GitHub Pages

### 4.1 一键部署命令（在项目根目录执行）

```bash
# Step 1: 构建
yarn webgal:build

# Step 2: 进 dist 目录初始化 git（仅第一次需要）
cd packages/webgal/dist
git init
git checkout -b gh-pages
touch .nojekyll
git remote add origin https://github.com/felixlovejj/WebGAL.git

# Step 3: 推送（每次更新都从这步开始）
git add -A
git commit -m "Update game"
git push -f origin gh-pages
```

> **注意**：如果你开了代理（如 Clash/V2Ray），push 时需要加代理参数：
> ```bash
> git -c http.proxy=http://127.0.0.1:7890 -c https.proxy=http://127.0.0.1:7890 -c http.postBuffer=524288000 push -f origin gh-pages
> ```

### 4.2 启用 GitHub Pages

1. 打开 `https://github.com/felixlovejj/WebGAL/settings/pages`
2. **Source** 选 `Deploy from a branch`
3. **Branch** 选 `gh-pages`，目录选 `/ (root)`
4. 点 **Save**
5. 等待 1-2 分钟，访问 `https://felixlovejj.github.io/WebGAL/`

> 之后每次 push 到 `gh-pages` 分支，GitHub Pages 会自动更新，无需手动操作。

## 五、日常更新流程（最简版）

每次改了剧本或加了素材后：

```bash
# 1. 构建
cd d:/project/WebGAL
yarn webgal:build

# 2. 部署
cd packages/webgal/dist
git add -A
git commit -m "Update game"
git push -f origin gh-pages
```

等 1-2 分钟刷新 `https://felixlovejj.github.io/WebGAL/` 即可看到更新。

## 六、常见问题

**Q: 为什么图片/音乐没显示？**
A: 检查文件名是否**完全一致**（包括大小写）且放对了目录。

**Q: 改完剧本刷新没变化？**
A: 本地预览（`yarn dev`）会实时更新。如果是 GitHub Pages 上的没变化，需要重新构建+推送。

**Q: 推送报 "connect to server" 错误？**
A: 代理没开或代理端口不对。检查 Clash/V2Ray 是否在运行，代理端口是否是 `7890`。

**Q: 立绘/背景图什么尺寸？**
A: 背景 2560×1440（16:9），立绘 1024×1536（竖版透明底），小头像 256×256。
